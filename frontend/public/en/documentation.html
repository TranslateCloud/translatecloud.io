Quick Start

Get started with TranslateCloud in minutes. This guide will walk you through translating your first website content.

1

Create an Account

Sign up for a free account at translatecloud.io/signup. No credit card required.

2

Get Your API Key

Navigate to Settings → API Keys and generate your first API key. Keep it secure!

3

Make Your First Translation

const response = await fetch('https://api.translatecloud.io/v1/translate', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    text: 'Hello, world!',
    target_language: 'es',
    preserve_html: true
  })
});

const data = await response.json();
console.log(data.translated_text); // "¡Hola, mundo!"

Installation

Install TranslateCloud using your preferred package manager:

npm 
npm install @translatecloud/sdk
yarn 
yarn add @translatecloud/sdk
CDN 
<script src="https://cdn.translatecloud.io/sdk/v1/translatecloud.min.js"></script>

Authentication

TranslateCloud uses API keys to authenticate requests. Include your API key in the Authorization header:

Authorization: Bearer YOUR_API_KEY
Keep your API keys secure

Never commit API keys to version control or expose them in client-side code. Use environment variables in production.

JavaScript SDK

The official JavaScript SDK provides a simple interface for all TranslateCloud features:

import TranslateCloud from '@translatecloud/sdk';

const client = new TranslateCloud({
  apiKey: process.env.TRANSLATECLOUD_API_KEY
});

// Translate text
const translation = await client.translate({
  text: 'Your website content',
  source: 'en',
  target: 'es',
  preserveFormatting: true
});

// Batch translate multiple strings
const batch = await client.translateBatch({
  texts: ['Home', 'About', 'Contact'],
  target: 'fr'
});

// Translate entire HTML documents
const html = await client.translateHTML({
  html: '<h1>Welcome</h1><p>Learn more</p>',
  target: 'de',
  preserveSEO: true
});

WordPress Plugin

Install the TranslateCloud WordPress plugin for seamless integration:

  1. Download the plugin from the WordPress repository or your TranslateCloud dashboard
  2. Upload and activate the plugin in WordPress Admin → Plugins
  3. Navigate to Settings → TranslateCloud and enter your API key
  4. Select target languages and configure SEO settings
  5. The plugin automatically detects and translates content

React Integration

Use the React hooks for dynamic translations:

import { TranslateProvider, useTranslate } from '@translatecloud/react';

function App() {
  return (
    <TranslateProvider apiKey={process.env.REACT_APP_API_KEY}>
      <MyComponent />
    </TranslateProvider>
  );
}

function MyComponent() {
  const { t, language, setLanguage } = useTranslate();

  return (
    <div>
      <h1>{t('welcome.title')}</h1>
      <p>{t('welcome.description')}</p>

      <button onClick={() => setLanguage('es')}>
        Español
      </button>
    </div>
  );
}

Translation Workflow

TranslateCloud supports multiple workflows to fit your needs:

1. Automatic Translation

Content is automatically translated as soon as it's published. Best for blogs and frequently updated sites.

  • Enable auto-translate in project settings
  • Configure quality threshold (AI confidence level)
  • Set up review notifications for low-confidence translations

2. Review & Approve

Translations require human review before publishing. Recommended for legal and marketing content.

  • Translations are generated but marked as "pending"
  • Review team receives notifications
  • Approve or request revisions through the dashboard

3. Manual Translation

Full control over translations. AI provides suggestions that translators can accept or modify.

  • Create translation projects manually
  • Assign to professional translators
  • AI suggestions speed up the process

SEO Optimization

TranslateCloud automatically handles SEO best practices for multilingual sites:

Hreflang Tags

Automatically generates proper hreflang annotations:

<link rel="alternate" hreflang="en" href="https://example.com/en/" />
<link rel="alternate" hreflang="es" href="https://example.com/es/" />
<link rel="alternate" hreflang="fr" href="https://example.com/fr/" />
<link rel="alternate" hreflang="x-default" href="https://example.com/" />

Meta Tags Translation

Translates and optimizes meta descriptions, titles, and Open Graph tags while maintaining SEO value.

URL Structure

Supports multiple URL patterns:

  • Subdirectories: example.com/en/, example.com/es/
  • Subdomains: en.example.com, es.example.com
  • Parameters: example.com?lang=es

Best Practices

1. Use Context-Rich Keys

Provide context in your translation keys for better accuracy:

// ✓ Good - provides context
t('nav.button.submit_form')
t('checkout.button.submit_payment')

// ✗ Avoid - ambiguous
t('submit')

2. Handle Pluralization

Use the built-in pluralization helpers:

t('items_count', {
  count: 5,
  // Automatically handles: "1 item" vs "5 items" in all languages
})

3. Test in All Languages

  • Some languages expand text by 30-40% (German, Russian)
  • Test UI layouts with all target languages
  • Use CSS to handle text overflow gracefully

4. Cache Translations

Improve performance by caching translations:

const client = new TranslateCloud({
  apiKey: process.env.API_KEY,
  cache: {
    enabled: true,
    ttl: 3600, // 1 hour
    storage: 'redis' // or 'memory'
  }
});

Error Codes

Code Description Solution

402 Payment required Upgrade your plan or add credits
429 Rate limit exceeded Slow down requests or upgrade plan
503 Service temporarily unavailable Retry with exponential backoff
